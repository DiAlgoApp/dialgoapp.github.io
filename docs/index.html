Documentación técnica

Documentación técnica

DiAlgo está diseñado como una plataforma modular sobre Firebase, con énfasis en seguridad y escalabilidad.

Stack principal

  • Astro + componentes UI (Preact).
  • Firebase Auth + Realtime Database.
  • Notificaciones push (Expo/Flutter).
  • Cloud Functions para lógica sensible.
  • CI/CD: GitHub Actions.
  • Hosting: GitHub Pages (Sitio estático).

Arquitectura

  • Frontend

    Sitio estático optimizado para SEO y accesibilidad.

  • Backend

    RTDB con reglas por rol y validaciones de datos.

  • Notificaciones

    Registro de tokens y envío controlado por funciones seguras.

  • Analytics

    Eventos clave para medir adherencia y coordinación.

API / Endpoints

Puntos clave

Endpoints internos orientativos. La API pública está en preparación.

Método Endpoint Descripción
POST /notifications/send Envía notificaciones push a un grupo autorizado.
POST /family/invite Genera invitaciones seguras para nuevos miembros.
GET /logs/:familyId Recupera historial auditable de eventos.
Modelo de datos

Estructura principal en RTDB

Colecciones y rutas principales para la coordinación de cuidados.

Rutas principales

  • families/{familyId}/members/{userId}
  • families/{familyId}/messages/{messageId}
  • families/{familyId}/medications/{reminderId}
  • families/{familyId}/logs/{logId}

KPIs recomendados

  • DAU / MAU y retención 7-30-90 días.
  • Mensajes enviados por semana.
  • Tasa de registro completado.
  • Permisos de notificaciones concedidos.
